# 80-QA.md — Quality Assurance Review

## Purpose
Perform a comprehensive review of the entire project before release.

## Input
- All previous phase outputs
- Complete source code

## Process

1. Review code quality and adherence to standards.
2. Run full test suite.
3. Perform security audit.
4. Check dependency vulnerabilities.
5. Verify documentation completeness.

## Output

```
docs/qa-report.md
```

## Template

```markdown
# QA Report

## Code Quality Review

### Standards
- [ ] TypeScript strict mode enabled
- [ ] No `any` types used
- [ ] ESLint passes with no errors
- [ ] Prettier formatting consistent
- [ ] No commented-out code
- [ ] No TODO/FIXME without tracking

### Architecture
- [ ] Folder structure matches architecture spec
- [ ] Components follow single responsibility
- [ ] Services are stateless where possible
- [ ] API error handling is consistent

## Test Results

| Test Suite | Total | Passed | Failed | Coverage |
|------------|-------|--------|--------|----------|
| Unit | 120 | 120 | 0 | 85% |
| Integration | 45 | 45 | 0 | 72% |
| E2E | 15 | 15 | 0 | - |

## Security Audit

### Dependency Scan
- Run: `npm audit`
- Critical: 0
- High: 0
- Medium: 2 (fix available)

### Security Checklist
- [ ] No secrets in codebase
- [ ] CSP headers configured
- [ ] Rate limiting enabled on auth endpoints
- [ ] Input validation on all API endpoints
- [ ] SQL injection prevention (Prisma ORM)
- [ ] XSS prevention (React auto-escaping)
- [ ] CSRF protection enabled

## Performance Check

- [ ] Lighthouse score > 90
- [ ] Bundle size within budget
- [ ] API p95 response < 200ms
- [ ] Database queries optimized

## Documentation

- [ ] README covers setup and development
- [ ] API documentation is current
- [ ] Environment variables documented in `.env.example`

## Conclusion

<!-- Pass/Fail, any blockers for release? -->
```

## Checklist

- [ ] Code quality review completed
- [ ] All tests passing
- [ ] Security audit clean (or risk accepted)
- [ ] Performance meets targets
- [ ] Documentation is complete
- [ ] QA report saved to `docs/qa-report.md`
